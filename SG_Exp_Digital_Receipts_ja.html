2022 年 8 月 20 日

このドキュメントの内容は、Concur Expense Standard Edition の新機能であり、以前は対象者限定設定ガイドで公開されていた e 文書タイムスタンプ機能と、以前はファクトシートで公開されていた領収書のデジタル化機能が含まれています。

デジタル領収書

注意: 複数の SAP Concur 製品バージョンおよび UI テーマを利用できるため、このコンテンツには、実装と正確に一致しないイメージまたは手順が含まれている場合があります。たとえば、SAP Fiori UI テーマが実装されている場合、ホーム ページのナビゲーションは [SAP Concur ホーム] メニューに統合されます。

概要

Concur Expense では、紙の領収書の代わりにデジタル領収書を使用する新規および既存のお客様のドキュメントが法律に遵守するようにデジタル領収書機能が提供されています。

この設定ガイドでは、以下のデジタル領収書機能について説明します。

  • e 文書タイムスタンプ (日本)
  • 領収書のデジタル化 (フランスおよびスペイン)

e 文書タイムスタンプ

日本のお客様は、e 文書タイムスタンプ機能によって紙の請求書の代わりにイメージを使用しながら e 文書が法律に遵守できます。

この機能は、日本の新規および既存のお客様 (および日本で従業員または構成を採用しているお客様) が使用できます。

要件

e 文書タイムスタンプ機能の要件は以下のとおりです。

  • 会社が日本の国パックを使用している必要があります。

タイムスタンプが設定された領収書の表示

エンド ユーザーの操作性

経費精算レポートを作成すると、タイムスタンプが正常に設定されたイメージとアイコン () がユーザーに表示されます。[経費に添付できる領収書] には、イメージごとに有効なタイムスタンプまたはステータスが表示されます。

  1. タイムスタンプが付けられたイメージに別のイメージを追加することはできません。
e 文書アイコン
e 文書タイムスタンプありの領収書

領収書イメージにタイムスタンプが正常に設定されると、領収書イメージの下部にタイムスタンプが表示されます。

e 文書タイムスタンプなしの領収書

タイムスタンプなしの領収書には、領収書イメージの下部に "不適格" と表示されます。

レポートの処理

処理者の操作性

十分なアクセス許可を有するユーザーが、タイムスタンプ フィールドを使って [レポートの処理] ページから照会を作成することができます。

  • チェック日付
  • 検証ステータス

この場合の十分なアクセス許可とは、以下のいずれかを意味します。

  • 経費の処理者
  • 経費処理者 (監査)
  • 経費処理者のマネージャー

詳細については、Concur Expense: レポートの処理設定ガイド (Standard Edition)をご参照ください。

タイムスタンプが設定された領収書の処理

タイムスタンプ ステータスのレビュー

タイムスタンプのステータスに応じて、領収書イメージの下部に以下のタイムスタンプ ステータスのいずれかが表示されます。

ステータス

説明

受領済

領収書がシステムに入力されました。

保留中

イメージがまだタイムスタンプの順番待ちに入っていません。

申請済

タイムスタンプが申請されました。

不適格

イメージが 3.88M ピクセル未満であるため、タイムスタンプに適格でありません。

注意: e 文書タイムスタンプ対象として適格になるには、イメージが 200dpi (スキャナ) または 3.88M ピクセル (スマートフォン/デジタル カメラ) を超えており、RGB が 24 ビットカラー以上である必要があります。

無効

イメージが検証チェックで不合格になりました。

  1. 検証ステータスはユーザーに表示されず、SAP Concur ではトラブルシューティングの目的でのみ使用できます。検証ステータスは、以下のとおりです。
    ● エンティティが見つかりません
    ● タイムスタンプが付いたイメージではありません
    ● 無効なイメージ
    ● 無効なイメージ ID
    ● イメージが見つかりません
    ● 削除済
    ● タイムスタンプをまだ受信していません
    ● 無効
    ● 有効

イメージの削除

以下のように、領収書イメージをユーザーが削除または添付解除できない場合があります。

  • [領収書ライブラリ] で、領収書イメージに正しくタイムスタンプが付けられているか、ステータスが保留中である場合、ユーザーは領収書イメージを削除できません。
  • 経費エントリ上で、領収書イメージに正しくタイムスタンプが付けられているか、ステータスが保留中である場合、ユーザーはエントリから領収書イメージの添付を解除できません。
  1. タイムスタンプのステータスが [不適格] や [無効] など赤字で表示されている場合は (ステータスが [保留中] でない限り)、領収書イメージの削除や添付解除が可能です。

構成

日本の国パックおよび日本向けの e 文書タイムスタンプ機能を使用する場合は、SAP カスタマー サポートにお問い合わせください。

デジタル化された領収書

領収書のデジタル化機能では、国の規制や政府当局に従って、紙の領収書を法定デジタル コピーに一致させる機能が SAP Concur のお客様に提供されます。

この機能は、フランスまたはスペインの新規および既存のお客様 (およびフランスまたはスペインで従業員または構成を採用しているお客様) が使用できます。今後のリリースで国が追加される可能性があります。

領収書デジタル化機能には、以下のようなものが含まれます。

  • スペインの税務局である Agencia Estatal de Administración Tributaria (AEAT) が監査およびホモロゲーション済のソリューションを提供します (スペインのみ)
  • 紙の領収書を公的価値を持つデジタル コピーに変換できます
  • Concur Expense の NextGen UI でのデジタル化された領収書の [認証済] ステータスの表示

スペインのホモロゲーション済ソフトウェアについては、AEAT の Web サイトをご参照ください。

フランスの法令については、Legifrance の政府の Web サイトをご参照ください。

領収書のデジタル化機能の SAP Concur 品質管理計画については、Concur Expense の担当者にお問い合わせください。

要件

領収書のデジタル化機能の要件は以下のとおりです。

  • 会社が Concur Expense ユーザー インターフェイス (UI) の NextGen UI を使用している必要があります。
  • 会社がフランスまたはスペインの国パックを使用している必要があります。
  • また、ユーザーが最新バージョンの SAP Concur モバイルアプリを使用している必要もあります。モバイルアプリでは ExpenseIt を使用できますが、ExpenseIt は必須ではありません。
  • 重要: ユーザーが Concur Expense の NextGen UI を使用していない場合は、SAP Concur でデジタル化された領収書のコンプライアンスを保証できません。

詳細については、Concur Expense の NextGen UI のガイドおよびリソースをご参照ください。

詳細については、領収書に関する SAP Concur モバイル アプリ ユーザー ガイド (領収書のデジタル化) をご参照ください。

詳細については、イメージ取り込みの最適化およびベスト プラクティスに関する ExpenseIt のトレーニング ビデオをご参照ください。

デジタル化された領収書について

領収書のデジタル化プロセスは、国固有のコンプライアンス ルールに従って紙の領収書に準拠しています。このプロセスが発生すると、領収書は [認証済] とみなされますこうした領収書は、他の領収書と同じように経費精算で使用できます。

  1. 紙からデジタルへの変換処理では、元々紙で発行された領収書のみがデジタル化されます。領収書を取り込み、変換するアプリケーションは、認証済ソリューションの一部でなければなりません。このため、これ以外の携帯アプリから写真を撮影し取り込んだ領収書や領収書ストア (未認証) に事前にアップロードされた領収書は、SAP Concur Mobile のデジタル化処理では適合化できません。
  2. 電子領収書は、電子ソースからの領収書原本とみなされ、領収書の各種規制および検証の対象になります。

用語集

用語

定義

認証済

Concur Expense に表示される、デジタル化された領収書に貼付されるラベル。

認証 / ホモロゲーション

本ドキュメントのコンテキストでは同じですが、ホモロゲーションの方がより具体的です (ホモロゲーションとは、公的認可機関による規制基準および仕様を製品が満たしていることを認証または承認する処理です)。

  • フランスでの紙からデジタルへの変換に対応するため、SAP Concur は Arrêté du 22 mars 2017 fixant les modalités de numérisation des factures papier en application de l’article L. 102 B du livre des procédures fiscales in France に準拠しています。
  • スペインでの紙からデジタルへの変換に対応するため、SAP Concur では税務局からのホモロゲーションの付与が必須です。ホモロゲーションは、Agencia Estatal de Administración Tributaria (AEAT - スペインの税務局) から付与されます。

コンプライアンス

行政機関から発行されたルールまたは規制に準拠すること。たとえば、領収書のデジタル化に関するルールへの SAP Concur のコンプライアンスなど。

適合化

紙の領収書の公的なデジタル コピーへの変換に関するそれぞれの国のルール / 規制を適用する処理。

デジタル化

紙の領収書を適合化した際の最終結果。適合化された領収書は “デジタル化“ されたものとみなされます。Concur Expense では、デジタル化された領収書に [認証済] というアイコンが付けられます。

ルール / 規制

紙の請求書 / 領収書の適合化およびデジタル化に関連する適用法令。

検証

データ検証のコンテキストにおいて、値を受け取り、内部ソースまたはサードパーティのソースに照らして検証するサービスまたは人。

各国の要件

紙の領収書や紙の領収書の公的コピーを取り込み、消費、保存、および取得するための SAP Concur ビジネスソリューションは現在、フランスとスペインで発行されたルールにのみ準拠しています。いずれの場合でも、それぞれの国や地域での準拠内容について、税務局や法務当局にお問い合わせください。

国

要件

フランス

ソリューションにより、ドキュメント複製の正確かつ完全な処理が保証されます。

  • 情報の損失がないことの保証。
  • フランスの法令
  • PDF 形式または PDF A/3 (ISO 19005-3)。
  • 変換されたファイルには、少なくとも RGS* の電子証明書に基づく電子署名が含まれている必要があります。証明書プロバイダは EU に属しているか、Trusted Services List (TSL) を使用している必要があります。
  • 電子コピーの作成日とドキュメントの識別情報を記録してください。
  • 変換後のドキュメントに対する変更の監査。

スペイン

ソリューションにより、ドキュメント複製の正確かつ完全な処理が保証されます。

  • 情報の損失がないことの保証。
  • スペインの法令
  • スペインのホモロゲーション プロセス
  • PDF 形式または PDF A/3 (ISO 19005-3)。
  • 変換されたファイルには、正規の電子証明書に基づく電子署名が含まれている必要があります。証明書プロバイダは、EU Trusted Services List (TSL) に属している必要があります。
  • 電子コピーの作成日とドキュメントの識別情報を記録してください。
  • 変換後のドキュメントに対する変更の監査。
  • スペインの税務局 Agencia Estatal de Administración Tributaria によるホモロゲーション ("認証")。

注意: ホモロゲーション済みソフトウェアについては、AEAT の Web サイトをご参照ください。

米国など、領収書の公的なデジタル コピーに関するルールが発行されていない国も多くあります。こうした国々では、ドキュメントが法律に遵守するように、それぞれの国の当局にお問い合わせになることを強くお勧めします。

紙の領収書の公的コピーの取り込み

紙の領収書の公的コピーを取り込むために、ユーザーはモバイル アプリ内で紙の領収書の写真を撮影します。写真が撮影されると、デジタル化処理がイメージに適用されます。このプロセスが完了すると、モバイル アプリでデジタル化された領収書にスタンプ が表示され、領収書が認証済であるかどうかを確認できます。

  1. 元の紙の領収書の法的コピーは、SAP Concur モバイル アプリでクイック経費を使用するか、経費精算レポート エントリを作成するか、ExpenseIt を使用して取り込むことができます。レポート ヘッダー レベルで取り込んだ領収書のデジタル化機能はまだ利用できません。

エンド ユーザーの操作性

Concur Expense の NextGen UI では、ユーザーの [レポートに追加できる経費]、[経費に添付できる領収書]、および経費リストにデジタル化された領収書が表示されます。これらのユーザーには、認証済の領収書に [認証済] アイコン が表示されます。

承認者と処理担当者の操作性

承認者および処理者には、Concur Expense の既存のバージョンの経費精算レポートおよび経費リストに [認証済] 領収書ステータスが表示されます。

デジタル化された領収書の処理

領収書のデジタル化が失敗した場合

デジタル化処理が中断された場合、イメージが認証されていない旨のメッセージがユーザーに表示され、もう一度試すよう指示されます。SAP Concur Mobile アプリのカメラが表示されるので、領収書の写真をもう一度取り込みます。

ExpenseIt を使用して写真を撮影する場合の技術的な失敗としては以下のようなタイプが含まれます。

  • 照明が薄暗かったり、カメラが遠すぎたりすることによる、ExpenseIt でのイメージの取り込み失敗
  • インターネット接続損失
  • サービス処理エラー

いずれの場合でも、写真を撮り直すか、処理が完了するまで待機するよう促すメッセージがアプリで表示されます。

詳細については、領収書に関する SAP Concur モバイル アプリ ユーザー ガイド (領収書のデジタル化) をご参照ください。

詳細については、イメージ取り込みの最適化およびベスト プラクティスに関する ExpenseIt のトレーニング ビデオをご参照ください。

デジタル化された領収書の削除

デジタル化された領収書の削除は現時点ではサポートされていません。ただし、デジタル化された領収書をレポート エントリから削除した場合、その領収書が経費エントリ内で SAP Concur Mobile アプリで取り込まれたものであると、添付されている認証済のイメージが削除されます。

デジタル化された PDF のダウンロード

領収書のデジタル化処理で作成された PDF には、証拠となる値を持つために必要なすべての情報が含まれます。経費精算レポートを開くためのアクセス許可を有するユーザーおよび管理者が、デジタル化された領収書が含まれる行項目に移動し、デジタル化された PDF ファイルをダウンロードできます。

各国のデジタル化された PDF ファイルの詳細については、本ドキュメントのデジタル化された領収書についてをご参照ください。

構成

フランスまたはスペインの国パックを使用し、Concur Expense の NextGen UI も使用または移行し、領収書のデジタル化機能を使用する場合は、SAP カスタマー サポートにお問い合わせください。

Concur Expense の NextGen UI への移行の詳細については、Concur Expense の NextGen UI 移行およびリソース (Standard Edition) をご参照ください。
