財務統合サービス (FIS)

セクション 1: アクセス許可

この機能へのアクセス権があるかどうかは場合によります。たとえば、特定のグループに対してのみアクセス権がある、または読取専用で作成や編集はできないなど、限定的なアクセス権を持っている場合があります。

管理者はこの機能を使用する必要がありますので、適切なアクセス許可がない場合は、社内の SAP Concur 管理者にお問い合わせください。

さらに、このガイドに記載されている作業は SAP Concur にのみ許可されているものもあります。必要に応じて SAP Concur クライアント サポートにご依頼ください。

セクション 2: 概要

財務統合サービス (FIS) を使用すると、外部システムを Concur Expense や Concur Invoice で生成された財務ドキュメントと連携させて、外部システムから財務システムに転記することができます。

このサービスでは、承認された経費精算レポート、仮払、請求書などの利用可能なデータ オブジェクトをリクエストするための自動化されたソリューションを提供します。オブジェクトをお客様の ERP にインポートした後、統合により転記確認を Concur Expense や Concur Invoice に戻すことができます。

Concur Expense や Concur Invoice から支出データを自動的に転送できることにより、以下が可能になります。

        エンドツーエンドでの支出の可視化が可能

        経費処理の迅速化により早急な支払が可能

        システム間の精度および同期が向上

        全自動の統合により処理にかかる時間およびリソースを軽減

        技術への確信ある投資判断が可能

財務統合 API

財務統合サービスは、財務統合 API に基づいています。経費精算レポートおよび請求書のシーケンス フロー など、API の詳細はすべて、財務統合 API 文書に記載されています。

 

 

財務データ オプション

Concur Expense および Concur Invoice では、財務仕分入力のデータを ERP に取り込むオプションは 2 つあります。

        財務統合サービス (FIS): API を介してデータが返されます。

        勘定抽出: バッチ スケジュールでフラット ファイルにデータが書き込まれます。

バッチ形式の勘定抽出ファイルよりも効率が良く、利点が多いため、可能な限り FIS を使用することをお勧めします。

財務統合サービスの利点

FIS には以下の利点があります。

SAP Concur ソリューションと ERP 間の一貫性の維持

FIS では、レポート、仮払申請、または請求書ごとに、ERP 連携のステータスを SAP Concur ソリューションに再度転記する必要があります。ERP で不具合が生じた場合、FIS はエラー メッセージを表示して、該当するレポート、仮払申請、または請求書を更新します。お客様は、処理者ツール内から修正アクションを開始します。このため、ERP に統合できるようになるまで、経費精算レポート、仮払申請、または請求書の処理は先に送られません。この処理は、ERP と SAP Concur ソリューション間のこの認識に慣れていない既存のお客様に対する変更です。

FIS を使用する Expense Pay のお客様は、ERP に正しく転記されたレポートについてのみ払戻を行います。逆に、勘定抽出処理を使用して処理したレポートについては、お客様の財務転記と関係なく、払戻金額が銀行に送信されます。

ERP へのリアルタイム統合

1 日を通して最終承認された支出データを取り込み、ERP に転記できます。1 日に一度のファイルベースの間隔を待つ必要がありません。

（Expense Professional のみ）支払確認

ERP パートナーの統合機能（またはお客様自身のアプリ）で、レポートの支払ステータスを Concur Expense に転記できます。

定義

財務文書: 経費精算レポート、請求書、および仮払申請は Concur Expense や Concur Invoice の財務文書に変換されます。財務文書は、買掛金勘定 (AP) エントリ、クーポン券、または取引に対応します。各財務文書に文書識別子 (ID) が割り当てられ、この ID を使用して ERP の文書が識別され、Concur Expense や Concur Invoice へ返されるフィードバックに取り込まれます。

通常、経費精算レポートごとに 1 つの財務文書が作成されます。統合により、1 つのレポートに対して複数の財務文書が作成される場合もあります（新規 GL、会社間処理）。

勘定抽出: 勘定抽出ファイルとは、レポート、請求書、または仮払申請の勘定情報が含まれるフラット ファイルです。通常、勘定抽出は、設定したスケジュールで 1 日に一度生成されます。このファイルは、お客様のエディションや設定によって異なります。

        Professional Edition: 勘定抽出は、標準勘定抽出 (SAE) ファイル、支払申請勘定抽出 (PRAE) ファイル、またはカスタマイズした抽出ファイルを示します。

        Standard Edition: 勘定抽出は、支払バッチ ファイル、または（まれに）カスタマイズした抽出ファイルを示します。

展開オプション

以下の 3 つの方法で FIS を利用できます。

1.    お客様が統合機能を構築する: このオプションでは、お客様が Concur Expense や Concur Invoice から財務システムに経費精算レポート、仮払申請、および請求書を転記する統合機能を作成します。お客様は FIS API を使用する前に、営業/会計管理部門と連携して、Client Web Services 契約を購入する必要があります。

2.    お客様が信頼できるいずれかの App Center パートナーを使用して統合機能を構築する: このオプションでは、お客様が信頼できるいずれかの App Center パートナーと連携します。パートナーは、Concur Expense や Concur Invoice からお客様の財務システムに経費精算レポート、仮払申請、および請求書を転記する統合機能を所有しています。お客様は、App Center パートナーと契約を結び、あらゆる統合設定で連携する必要があります。

3.    SAP Concur のテクニカル イネーブルメント チームが統合機能を構築する: このオプションでは、お客様が SAP Concur のテクニカル イネーブルメント チームと連携します。SAP Concur のテクニカル イネーブルメント チームがお客様の要件に基づいてカスタムな統合機能を構築します。パートナーは、Concur Expense や Concur Invoice からお客様の財務システムに経費精算レポート、仮払申請、および請求書を転記する統合機能を所有しています。お客様は営業/会計管理部門と連携して、Hosted Customer Connector を契約に追加する必要があります。

新規のお客様

Concur Expense および Concur Invoice の新規のすべてのお客様では、勘定抽出ではなく、FIS データの使用が推奨されます。当社の販売組織と連携して、FIS を利用するためのいずれかのオプションを選択することができます。

&   有効化の詳細については、本ガイドの「設定」および「お客様側の計画」セクションをご参照ください。

既存のお客様

まず、勘定抽出ファイルをデータ ソースとして使用しているお客様のすべての統合を特定します。その後、FIS データでお客様のすべての勘定抽出ファイルの統合機能に置換できるかどうか確認します。ほとんどのお客様は勘定抽出を仕分入力にのみ使用しているものと思われます。この場合、代わりに FIS を使用できます。お客様によっては、分析データベースに勘定抽出を使用している場合があります。このような場合は、FIS に加えて、情報抽出が必要となります。

        FIS フロー用に設定されているレポートは勘定抽出に含まれません。

        FIS フローで有効になっていないレポートは、引き続き SAE、PRAE、または支払バッチ ファイルに含まれます。

&   詳しくは、本ガイドの「設定」および「お客様側の計画」セクションをご参照ください。

セクション 3: お客様側の計画

処理の一貫性を保つために、すべてのお客様がすべての経費グループまたは請求書ワークフローに対して FIS を有効化することをお勧めします。

お客様の ERP でファイルが必要な場合、統合機能の開発者は FIS データに基づきファイルを作成することができます。これにより、お客様は 1 つの一貫した方法を使用して、Concur Expense および Concur Invoice から財務データを取り込むことができます。

（Professional Edition のみ）情報抽出

お客様は FIS を有効化するだけでなく、何らかの情報抽出を作成する必要があるかどうか確認します。情報抽出は財務統合目的には使用しないでください。重複を排除するための制御レベルが欠如しています。情報を転記するために信頼できるソースを 1 つ（勘定抽出または FIS 転記文書のいずれか）に絞る必要があります。お客様のその他の統合ニーズを満たすデータを FIS で作成できない場合にのみ、情報抽出を作成してください。情報抽出の例として、以下のようなものが挙げられます。

        同席者の詳細抽出: コンプライアンス レポートの目的に使用されます。ライフ サイエンスのお客様の場合、情報抽出として作成された「同席者の詳細」抽出ファイルが必要になる可能性があります。

        Expense Pay 確認抽出

        標準勘定抽出 (SAE) や支払申請勘定抽出 (PRAE): 財務仕分入力用ではありません。ほとんどのお客様の場合、FIS の使用に加えて、これらのファイルの作成が必要となることはありません。

固有の FIS 統合機能をお使いの場合は、SAP Concur と連携して、お客様の要求に基づき情報抽出を設定する必要があります。既存のお客様は、SAP Concur サポートにサポート チケットを送信してください。App Center パートナーと連携している場合は、パートナーが上記のいずれかのファイルを情報抽出として引き続き使用するためのケースを送信します。

既存のお客様に対する変更管理

FIS が有効な場合、レポートや請求書が勘定抽出に送られなくなることを認識しておくことが重要です。あらかじめ移行期間の対処方法を決めておいてください。

        既存の勘定抽出と FIS の並列処理を実装する

- または -

        FIS を有効化する前に、処理途中のすべてのレポートおよび請求書を処理する

最初のオプションでは、お客様は既存の勘定抽出処理を維持する必要があります。FIS が有効化される前に作成されたレポートや請求書は引き続き勘定抽出に送られます。作成日が FIS の有効化以降であるレポートや請求書は FIS 処理に送られます。すべてのレポートや請求書が ERP で最終処理されるまで、既存の勘定抽出の統合を維持する必要があります。

2 番目のオプションを使用する場合は、FIS を有効化する前に、処理途中の経費精算レポート、仮払申請、または請求書が残っているユーザーがないことをまず確認する必要があります。「処理途中」とは、以下のようなレポート、仮払申請、または請求書を指します。

        作成済であるが、未提出である

        提出済であるが、承認ワークフローが完了していない

この場合、移行を管理する上で、以下のようなツールが有用です。

        対応付けスプレッドシート - 抽出ファイルと FIS の対応付けを確認するには、標準勘定抽出 (SAE) ファイルや支払申請勘定抽出 (PRAE) ファイルをご参照ください。

        （Expense のみ）処理者検索ツール - 勘定抽出に送るレポートを識別するためのクエリを作成します。処理者クエリには、「財務統合」 = No という条件に加え、「承認済および経費審査中」といったお客様が必要とする条件を指定します。このクエリでは、FIS が有効化される前に作成されたレポートが返されます。

        （Expense のみ）監査ルール FIS への切り替え中に「レポートをコピー」機能の使用を禁止する監査ルールを作成できます。コピー機能で FIS に新規レポートが送信されなくなるため、従業員が既存のレポートでこの機能を使用しないようにしてください。「レポートをコピー」機能は、FI がすでに有効化されているレポートに使用できます。このルールの作成については、サポート部門または実装部門が支援します。

作成日と FIS フラグを使用して、FIS が有効化される前に作成されたレポートをユーザーがコピーしないようにする監査ルールの例を以下に示します。

さらに、お使いのバージョンに応じて留意すべき相違点がいくつかあります。

Standard Edition

        勘定抽出に設定されたレポートがある場合に限り、支払マネージャーのバッチに [ファイルのダウンロード] リンクが表示されます。これにより、それまで支払バッチ ファイル方法を使用していて FIS 方法に移行した場合、それまでの勘定抽出ファイルを取り出すことができます。

        FIS 方法が有効化された後に作成されたレポートはすべて FIS 処理に回されます。これは、Standard Edition のグローバルな設定です。処理途中のレポートは、処理途中のすべてのレポートが処理されるまでは、それまでの支払バッチ ファイル方法で処理されます。

Professional Edition

        会社の設定に応じて、勘定抽出に設定されたレポートがある場合に限り、支払マネージャーのバッチに [ファイルのダウンロード] リンクが表示されます。

        経費グループに応じて、従来の勘定抽出方法を使用するよう設定したり、FIS を使用するように設定することができます。これは、経費グループ レベルで制御されます。

        請求書処理グループに応じて、従来の勘定抽出方法を使用するよう設定したり、FIS を使用するように設定することができます。これは、請求書ワークフロー レベルで制御されるため、請求書処理ポリシー/請求書処理グループ別になります。

セクション 4: 経費精算ユーザーへの表示

Concur Expense および Concur Invoice では、エンド ユーザーに対していくつかの箇所で統合機能が表示されます。

        経費精算レポートおよび仮払の監査証跡

        [レポートの処理] ページ

        [仮払申請管理] ページ（Professional Edition のみ）

経費精算レポート/仮払の監査証跡

統合により転記フィードバックが返されると、レポート/仮払の監査証跡に転記ステータスが記録されます。[詳細] > [監査証跡] をクリックすると、監査証跡が表示されます。

レポートの処理ページ

経費の処理者は通常の手順で支払レポートを承認します。Professional Edition では、ユーザーの経費グループによって、 FIS で支払が行われるレポートが決まります。Standard Edition では、機能の有効化後に作成されたすべてのレポートにこの機能が適用されます。

承認後、財務転記の順番待ちに経費精算レポートが入れられます。統合のタイミングおよび頻度は、統合を実施するお客様が設定可能です。財務転記が完了すると、FIS API を使用して転記フィードバックが送られ、処理者は [レポートの処理] ページでフィードバックを確認できます。

正常に転記されたレポートには支払ステータス「転記済/支払送信」（設定に応じて異なる）が表示され、監査証跡に財務転記の詳細が記録されます。監査証跡メッセージには、お客様の財務システムから送信された情報が取り込まれます。

転記に失敗したレポートでは、規定外フラグおよび支払ステータス「財務転記失敗」が表示されます。

処理者はレポートの監査証跡を表示して、転記エラーの詳細を確認できます。

処理者は修正を行うため、レポートを撤回します。修正を行ったら、FIS の順番待ちに再度入れることができます。正常に転記されると、支払ステータスが「支払送信」または「支払済」（設定に応じて異なる）に変更された旨のフィードバックが送信されます。

NOTE: 財務転記の順番待ちに経費精算レポートが送信されると、ユーザーのプロファイルから従業員情報が直接取り込まれます。経費精算レポート ヘッダーまたは経費エントリ フィールドにコピー ダウンされた従業員データは使用されません。

財務統合有効列

[リストの設定] オプションを使用して、[財務統合有効] というラベルの列を [レポートの処理] ページに追加できます。

有効化されると、財務統合サービスを使用して処理されるように設定されているレポートがこの列に示されます。

財務転記失敗クエリ

Concur Expense では、統合機能をお使いのお客様専用のクエリ「財務転記レポート失敗」が [レポートの処理] ページに表示されます。処理者はこのクエリを使用することで、支払ステータスが「財務転記失敗」であるすべてのレポートを素早く見つけることができます。

（Professional Expense のみ）支払確認ステップ

Concur Expense Professional Edition をお使いのお客様は、支払確認ワークフロー ステップを使用して、さらに更新を行うこともできます。Concur Expense で FIS が有効化されると、レポート支払先にはお客様の財務システムによって支払が行われるため、レポートの監査証跡に付加情報が記録されます。すべてのレポート支払先に支払が行われると、レポートは「支払確認済」支払ステータスになります。 

（Professional Expense のみ）仮払申請管理

仮払申請管理者は通常の手順で仮払を作成および発行します。[仮払申請管理] ページの [仮払申請ステータス] リストに「仮払申請失敗」ステータスのオプションが追加されました。

財務統合が有効になると、ページに新しい [FI ステータス] 列が表示されます。

「失敗」ステータスの仮払申請の場合、仮払申請管理者は仮払申請の一部のフィールドを変更できます。仮払申請名をクリックすると、詳細が表示されます。

仮払申請管理者はフィールド値の間違いの修正後、[FI へ差戻す] ボタンをクリックして、財務統合に仮払申請を再送信できます。

セクション 5: 請求書処理ユーザーへの表示

Concur Invoice では、エンド ユーザーに対していくつかの箇所で統合機能が表示されます。

        請求書の監査証跡

        [申請の処理] ページ

請求書の監査証跡

統合により転記フィードバックが返されると、請求書の監査証跡に転記ステータスが記録されます。[詳細] > [監査証跡] をクリックすると、監査証跡が表示されます。

申請の処理ページ

請求書の処理者は通常の手順で支払申請を承認します。Professional では、ユーザーの請求書処理グループによって、 FIS で支払が行われる申請が決まります。Standard では、機能の有効化後に作成されたすべての請求書にこの機能が適用されます。

承認後、財務転記の順番待ちに申請が入れられます。財務転記が完了すると、処理者は [申請の処理] ページで転記フィードバックを確認できます。

財務転記失敗クエリ

処理者はカスタムな処理者クエリ ツールで [申請] データ オブジェクトおよび [支払ステータス] フィールドを使用して、ステータスが「財務転記失敗」のすべての申請を検索できます。

ワークフローの処理者ステップの確認

転記エラー時に、処理者が「財務転記失敗」ステータスのレポートを確認できるようにするため、処理者ステップがワークフローの一部になっている必要があります。これにより、処理者はレポートを撤回して、修正を行うか、またはエンド ユーザーに差戻すことができます。このワークフロー ステップはスキップ ステップにできますが、ステップを省略することはできません。

勘定抽出と比べて FIS の大きな利点の 1 つは、ERP と SAP Concur システムを同期させることができる点です。 この同期化は、既存のお客様にとって変更点です。経費の処理者は、処理者クエリを使用して、失敗したレポートを探し、修正または差戻す必要があります。

&   詳しくは、本ガイドの「経費精算ユーザーへの表示」 > 「レポートの処理ページ」セクションをご参照ください。

一般的な使用例

使用例 1: ユーザー → 処理者 → 上長

このケースでは、レポートが失敗すると、処理者の順番待ちに表示されます。処理者がレポートを再度承認すると、転記の前に、承認を求めて上長に送られます。

使用例 2: ユーザー → 上長 → 処理者 

これが理想的なワークフローです。このケースでは、レポートが失敗すると、処理者の順番待ちに表示されます。処理者は失敗したレポートを再度開き、修正後、再送信します。これで、レポートが転記できる状態になります。

セクション 7: よくある質問

FIS を有効にするには、どうしたらいいですか。

本ガイドの「概要」 > 「展開オプション」をご参照ください。

        新規のお客様: 選択したオプションによっては、さらに購入が必要になる場合があります。

        Client Web Services をお使いの既存のお客様: SAP Concur サポートにサポート チケットを送信してください。詳しくは、本ガイドの「設定」 > 「有効化のためのサポート チケットの送信」セクションをご参照ください。

        Client Web Services をお使いでない既存のお客様: 割り当てられている SAP Concur アカウント マネージャーにお問い合わせください。さらに購入が必要になる場合があります。

すでに Web サービスが契約に含まれている場合、FIS API をすぐに使用できますか。

SAP Concur 管理者にのみ表示される設定ステップがあるため、SAP Concur に FIS の有効化を依頼する必要があります。サポート問合せ権限の保持者 (ASC) がサポート チケットを発行して、財務統合サービスの有効化を依頼してください。

&   詳しくは、本ガイドの「設定」 > 「有効化のためのサポート チケットの送信」セクションをご参照ください。

勘定抽出と FIS の両方を有効にすることはできますか。

Professional Edition をお使いのお客様は、勘定抽出と FIS を同時に有効化できます。Standard Edition をお使いのお客様の場合は、勘定抽出と FIS を同時に使用することはできません。

有効化後、以前の経費精算レポートは FIS の順番待ちに表示されますか。

FIS が有効化された後に作成されたレポートのみが FIS の順番待ちに表示されます。それ以前のレポートは勘定抽出に送られます。

FIS には、経費精算 (sae) または請求書処理 (prae) で利用可能なすべてのフィールドが含まれていますか。

お客様用の比較文書に FIS で使用可能なフィールドがリストされているので、ご確認ください。

&   詳しくは、本ガイドの「お客様側の計画」 > 「既存のお客様に対する変更管理」セクションをご参照ください。

FI のサンプル文書または使用ケースはどこで参照できますか。

詳細については、財務統合 API 文書をご参照ください。

FIS はどの位の頻度で呼び出すことができますか。

FIS を呼び出すスケジュールについては、お客様に管理が任されています。間隔を 10 分以上空けたスケジュールが推奨されます。
