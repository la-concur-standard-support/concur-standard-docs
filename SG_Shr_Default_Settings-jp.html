既定の構成設定

概要

Expense、Invoice、および Request には、いくつかの重要な既定の構成設定があります。ただし、これらの設定を表示または調整することはできません。

Expense / Invoice の既定の設定

既定の設定は、メール通知、備忘メール、設定、およびユーザーのアクションに基づいて発動するその他の設定不可の機能で構成されています。

Standard 製品に実装されると、その他の既定の設定が追加されます。

備忘メールとメール通知

Expense の期限切れの取引、または Invoice の期限切れの申請に対して、備忘メールやメール通知が送信されるタイミングまたは条件を除き、この表のオプションは下記の既定の設定になり、変更はできません。

Expense / Invoice のデジタル税請求書　※日本未対応

Expense または Invoice をお使いの会社で利用可能な「デジタル税請求書」機能では、サポート対象国（現在はメキシコ）のユーザーが、デジタル税請求書（XML で）を経費または請求書に添付できます。特定の国では、払戻し可能なすべての経費に対して、または支払先に支払うことができるように、この税請求書が必要になります。経費の払戻しや支払先への支払を行う前に、サードパーティの税請求書バリデータ（検証ツール）で税請求書を確認する必要があります。

税請求書が経費精算レポートまたは請求書にアップロードされると、デジタル税請求書の Web サービスで、サードパーティのバリデータが Concur プラットフォームを使用して税情報を表示および検証できます。バリデータは、Web サービスを使用して結果を Concur に送信します。

メキシコでは、Comprobante Fiscal Digital、または CFDi 形式が必要です。この機能は現在この形式をサポートしていますが、将来的に拡張され追加の形式をサポートする可能性があります。Concur は、デジタル税請求書からの UUID 情報を保存します。

メキシコでは、支払先が払戻し可能な経費や支払対象の請求書に対して XML 形式のデジタル税請求書を提供することを義務付けられています。会社は、税目的で経費を請求するためにこのファイルを受け取って保存し、監査の場合に元のファイルを作成する必要があります。この要件を満たすために、ユーザーは公式のデジタル税請求書の XML ファイル（Comprobante Fiscal Digital、または CFDi）を経費精算レポートまたは請求書にアップロードして保存できます。

この機能では、次のことが可能です。

  • エントリまたは請求書の CFDiフ ァイルのアップロード
  • CFDi アイコンにカーソルを合わせてデータを表示
  • お好みの XML ビューアーで元の CFDi を表示
  • エントリまたは請求書から CFDi を削除

メキシコの国別パックが Concur Standard をお使いの会社に対して有効になった後、CFDi ファイルをアップロード、表示、および削除する機能がすべての従業員に自動的に提供されます。会社にメキシコの国別パックがある場合、ユーザーには [CFDi を添付] ボタン（Expense の場合）または [CFDi を添付] コマンド（Invoice の場合）が表示されます。その他の設定は必要ありません。

経費精算レポート

デジタル税請求書機能では、従業員に対して [経費エントリ] ページに [CFDi を添付] ボタンが表示されます。ユーザーはボタンをクリックして CFDi ファイルに移動し、アップロードするファイルを選択します。デジタル税請求書が保存されると、エントリの横に青い CFDi アイコンが表示されます。ユーザーはこのアイコンにカーソルを合わせて CFDi 情報を表示するウィンドウを開き、CFDi を開く（ユーザーの既定の XML ビューアーを使用）か、エントリから削除することができます。

ユーザーには、経費エントリを開くときに [CFDi を添付] ボタンが表示されます。

  2. 同じ CFDi の XML ファイルが複数の経費エントリに添付されている場合、Expense は規定外フラグを表示します。
  3. CFDi ファイルを添付するには：
  4. 目的の経費エントリを開きます。
  5. [CFDi を添付] をクリックします。
  6. [CFDi のアップロード] ウィンドウでファイルを探して選択します。
  7. [アップロード] をクリックします。

  1. エントリが保存され、[費用] の下のエントリの横に CFDi アイコンが表示されます。

請求書

「デジタル税請求書」機能では、従業員に対し、請求書の [アクション] メニューに [CFDi を添付] および [CFDi を削除] コマンドが表示されます。ユーザーはオプションを選択して CFDi ファイルに移動し、アップロードするファイルを選択します。デジタル税請求書が保存されると、請求書に CFDi アイコンが表示されます。ユーザーはこのアイコンにカーソルを合わせて CFDi 情報を表示するウィンドウを開き、CFDi を開く（ユーザーの既定の XML ビューアーを使用）か、請求書から削除することができます。

有効にすると、[アクション] > [CFDi 添付ファイル] > [CFDi を添付] をクリックして、請求書を作成または編集できます。

  1. 1 つの請求書につき 1 つの CFDi ファイルが添付できます。そのため、CFDi がすでに申請に添付されている場合、[CFDi を添付] オプションは表示されません。ユーザーは、現在添付されている CFDi ファイルを削除して、別の CFDi ファイルを添付する必要があります。
  2. CFDi ファイルをアップロードするには：
  3. 請求書を作成するか、既存の請求書を選択して開きます。
  4. [アクション] > [CFDi 添付ファイル] > [CFDi を添付] をクリックします。

  1. [CFDi を添付] ウィンドウをクリックして CFDi ファイルを探し、[アップロード] をクリックします。

これで、CFDi ファイルが請求書に関連付けられました。以前にアップロードされた CFDi ファイルがある場合、アップロードの操作を行うと既存のファイルが置き換えられます。

請求書を保存すると、CFDi アイコンが表示されます。

  1. CFDi ファイルを削除するには
  2. 目的の請求書を開きます。
  3. [アクション] > [CFDi 添付ファイル] > [CFDi を削除] をクリックします。

または、ユーザーは、請求書の CFDi アイコンの上にマウス ポインターを合わせて、[CFDi を削除] をクリックします。

Expense: 支払マネージャー抽出の変更（任意）

抽出に追加できる 2 つのフィールドは次のとおりです。

  • レポート エントリの XML 領収書 UUID
  • レポート エントリの XML 領収書サプライヤー税 ID
  • 詳細については、設定ガイド Standard Edition「Expense -ファイル エクスポート構成」またはユーザー ガイド Standard Edition「Expense - 支払マネージャー抽出」をご参照ください。

サードパーティ検証

デジタル税請求書にサードパーティの検証サービスを使用する場合は、バリデータと契約できます。バリデータが Concur のデジタル税請求書 Web サービスを使用している場合、デジタル税請求書に直接アクセスして、検証ステータスを更新できます。バリデータと契約する必要があります。Concur の認定検証パートナーは、App Center または https://www.concur.com/en-us/app-center で確認できます。

サードパーティの検証サービスが使用されている場合、処理者に未検証で有効かどうか確認されていない XML ファイルが添付された経費の規定外フラグが表示されます。この規定外フラグでは、規定外フラグ コード INVXMLST を使用するため、この規定外フラグを含むレポートまたは請求書に処理者のレビューが必要であることをワークフロー ルールに記載できます。規定外フラグのあるレポートまたは請求書は、常に処理者に送られて確認されます。

処理者は、青色の CFDi アイコンにカーソルを合わせると表示される情報内で、デジタル税請求書に関するステータス情報を確認できます。ステータスには次のようなものがあります。

  • 空白（未検証）
  • 有効
  • 有効（警告あり）
  • 無効

バリデータには、ステータスが「有効」（警告あり）または「無効」であるファイルについて、どのアクションを実行するのが適切かについて処理者が参考にできる説明コメントが含まれます。

Request の既定設定

Request に基づく設定

Request 製品は現在、出張ベースの申請をサポートしています。古い申請は Concur によって自動的に終了されます。終了日を決定するため、Concur は申請の終了日に 90 日を加えた日に申請を自動的に終了します。申請に終了日がない場合、最終承認日を使用するか承認後にキャンセルされた日を使用します。
